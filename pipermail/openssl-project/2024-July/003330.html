  Add Doxygen comment requirement to coding style

In an effort to better document our code in such a way that is condusive
to future developers getting involved with our code base, I thought I
would propose adding a requirement to our coding style that
structures/macros/functions/etc should have Doxygen style comments added
to them, so that we could build alternate docs directly from our source
base.
