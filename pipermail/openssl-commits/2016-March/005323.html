    Correct slight logic error in processing IF in build.info
    
    This corrects a fault where the inner IF in this example was still
    being acted upon:
    
      IF[0]
        ...whatever...
        IF[1]
          ...whatever more...
        ENDIF
      ENDIF
    
    With this change, the inner IF is skipped over.
