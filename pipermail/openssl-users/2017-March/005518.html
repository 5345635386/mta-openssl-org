On 29/03/17 09:43, Jeffrey Walton wrote:
> Any ideas how to get OpenSSL to use /usr/local as a prefix, but place
> its libraries in /usr/local/lib64?

libdir is relative to prefix, to just use "--libdir=lib64"

>From INSTALL:

  --libdir=DIR
                   The name of the directory under the top of the
installation
                   directory tree (see the --prefix option) where
libraries will
                   be installed. By default this is "lib". Note that on
Windows
                   only ".lib" files will be stored in this location.
dll files
                   will always be installed to the "bin" directory.
