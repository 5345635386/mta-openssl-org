I built the FIPS capable OpenSSL in the standard way (i.e. ./config fips; make;
make install) and it worked.

After some tracing on the source code of fips.c I found that the mismatch
error was due to the fact that the content of the two variables "sig" and
"FIPS_SIGNATURE" are different when the OpenSSL lib was built using the
Ubuntu build script. When building the lib using the standard way, the
content of "sig" and "FIPS_SIGNATURE" are the same.

Does anyone know what could cause the content of "sig" and "FIPS_SIGNATURE"
 to be different (and hence causes the "fingerprint does not match" error)?

Thanks and any suggestions are truly appreciated.
