On Sat, Oct 03, 2015 at 07:01:50PM +0530, Mitesh Thakkar wrote:

> In the first try of connection it is working well and gets response from
> web service. In the second try onwards it gives error -
> SSL_R_UNABLE_TO_LOAD_SSL2_MD5_ROUTINES
> all the time.

Something ran OpenSSL library cleanup after the first call and
deregistered all the digests.

If it is not your application, then it the library that uses OpenSSL
on your behalf.

The failure is in SSL_CTX_new(), which should typically be done
just once, and re-used for a second connection.  Clearly you're
calling it again, and this type with the OpenSSL library already
"finalized".
