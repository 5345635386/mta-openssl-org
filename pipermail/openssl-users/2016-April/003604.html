I don't know why that message wasn't answered, because it's a well-known issue. I would have mentioned it in my previous note but it slipped my mind.

You get that error when you try to use the Microsoft assembler (ml.exe or ml64.exe). For OpenSSL on Windows, you now must use nasm, which is an open-source assembler. You can get it at www.nasm.us<http://www.nasm.us>. Just download it, install it (which I think is just unzipping an archive), and put it in the path.
