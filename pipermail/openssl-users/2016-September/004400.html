We started using OpenSSL in 2010 for Windows and Linux. We gladly followed the release strategy suggestion, as it allowed us to deliver patches without recompiling our code. So we still compile and link our code against 1.0.0 while delivering the current 1.0.2x versions (which by the way works great).
At Windows we started with VC10 and soon migrated to VC12, but currently we are struggling with porting to VC14. So we are considering to update the openSSL version we compile and link against to 1.0.2. I don't expect 1.0.2 to be binary compatible to 1.0.1 or 1.0.3 as 1.0.0 is, but I expect it to be binary compatible with all 1.0.2x versions. Since I found no official statement which confirms this, I'd ask you to confirm this theory of mine.
